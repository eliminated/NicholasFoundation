Nyx AI

Core Capabilities

Intelligence
From the Ground Up

💬

Rule Engine Chat

Normalized token and phrase matching against JSON-driven intent configs. Priority-based scoring with multi-signal ranking — trigger length, token count, and config order — for precise, deterministic responses.

🧠

Persistent Memory

Structured fact storage in memory.json with conversation transcript logging. Supports name recall, memory export, selective forgetting, and recent-turn retrieval across sessions.

📋

Full Organizer

Built-in notes with folders and tags, task management with completion tracking, and timed reminders with natural-language scheduling — stored locally in JSON and SQLite.

🖥️

WPF Desktop GUI

A polished C# WPF app with Chat, Memory, Organizer, Transcript, Usage, Settings, Tuning, and About pages. Communicates with the C++ engine via a JSON-line IPC protocol.

🔔

Reminder Service

A Windows Service (Nyx.ReminderService) with SQLite-backed reminder scheduling, localhost HTTP API, optimistic concurrency, and a per-user tray notifier process for delivery.

🛡️

Safety & Moderation

Category-based safety intents with priority escalation for harmful and victim scenarios. Guided recovery flows for account security. Safety intents always outrank casual chat.

Multi-Layer
Architecture

Integration

JSON-Line
Protocol

📨

Typed Requests

13+ typed request types including chat, version, config_snapshot, organizer snapshots/mutations, reminder service status, diagnostics, and shutdown — all with structured envelopes.

🔧

Safe Tuning

Live intent trigger/response editing via save_config, with automatic validation, timestamped backup, and engine restart — IDs, categories, and priorities stay locked.

🔄

Backward Compatibility

Plain text and legacy JSON both gracefully degrade to chat. Protocol version 1 guarantees stable envelope structure across CLI, GUI, and future clients.

📊

Usage Metering

Local counters for total turns, daily turns, and session turns. No external telemetry — all analytics are kept on-device and fully user-controlled.

The Future

From Rules
to Machine Learning

Nyx is currently rule-based, but the architecture is designed to evolve. The goal: build a competitive AI model from the ground up.

✅

Phase 1 — Rules

Rule engine with normalized matching, intent priorities, guided flows, safety categories, and persistent memory. Desktop GUI with full organizer. Current stage.

🔬

Phase 2 — ML Integration

Introduce lightweight machine learning models for intent classification, context-aware responses, and sentiment detection — while preserving the deterministic safety layer.

🚀

Phase 3 — Competitive AI

Scale to a fully competitive conversational AI — aiming to match the quality and capabilities of leading models like Claude and ChatGPT.
